# 화면별 구현 계획

## 1. 시장종합

- **수집 주기**: 장중 5분 (현재 collectMarketData에 포함) — 현행 유지
- **저장 전략**: latest (MarketOverview) + snapshot (MarketOverviewSnapshot)
- **수집 단위**: 코스피·코스닥 각 1회 (inds_cd: 코스피=001, 코스닥=101)
- **현황**: Collector 구현 완료, 경로 수정 완료
- **미결 사항**
  - 응답 필드 중 `pred_pre`(전일대비 변동값), `mrkt_stat_cls_code`(시장상태) 실제 필드명 테스트 확인 필요

---

## 2. 투자자별매매종합

- **수집 주기**: 장중 5분 (현재 collectMarketData에 포함) — 현행 유지
- **저장 전략**: latest (InvestorTradingSummary) + snapshot
- **수집 단위**: 코스피·코스닥 각 1회
- **처리 방식**: ka10051 응답은 전체 업종 목록이므로, 응답에서 inds_cd=001(코스피) 또는 101(코스닥) 항목 필터링
- **현황**: Collector 구현 완료, 경로·base_dt 수정 완료
- **미결 사항**
  - ka10051은 순매수 금액만 제공 (매수/매도 분리 없음) — 화면 설계 시 이 점 반영 필요
  - 실제 응답에서 종합지수 inds_cd 값(001/101) 테스트 확인 필요

## 3. 장중투자자별매매상위

- **수집 단위**: (시장 2 × 투자자 3 × 매매방향 2) = 12회 API 호출

# 현재 코드 구조상 문제점

## 1. IntradayInvestorRankingCollector 스케줄러 위치 오류 (높음)

현재 장중 5분 배치(`collectMarketData`)에 포함되어 있으나, ka10065 데이터는 18:30 이후에만 제공된다.
장중에 호출하면 매 5분마다 빈 응답 또는 오류 발생. 18:30 이후 단독 배치로 분리 필요.

## 2. 응답 래퍼 필드명 전체적으로 불확실 (높음)

`MarketOverviewCollector`는 `response.path("output")`으로 읽는데, ka20001이 실제로 `output`으로 감싸는지 미확인.
필드명이 틀려도 오류가 나지 않고 값이 0으로 조용히 저장된다. 테스트 없이는 실제로 데이터가 들어오는지 알 수 없음.

## 3. ka90008 응답 배열 순서 불확실 (높음)

현재 `outputList.get(0)`으로 첫 번째 항목만 저장.
API가 최신 시각 기준 내림차순이면 맞고, 오름차순이면 매 수집마다 당일 09:00 데이터만 저장하는 구조가 된다.
응답 순서를 모르는 상태.

## 4. ka90013 수집 코드 없음 (중간)

일별 프로그램매매추이 화면 지원 예정이나 Collector에 ka90013 호출 로직이 없고,
스케줄러에 장외 배치도 없음.

## 5. ProgramTradingCollector 코스닥 미수집 (중간)

`mrkt_tp = "P00101"` 코스피만 수집 중. 코스닥(`P10102`) 누락.

## 6. ProgramTradingHistory 중복 저장 방어 없음 (낮음)

ShortSellingCollector는 `existsByStockCodeAndTradeDate` 체크가 있으나,
ProgramTradingHistory는 동일 (stockCode, snapshotTime) 조합 중복 저장 방어 로직 없음.

## 7. IndexContributionRankingCollector 동작 불가 상태 (별도)

추정값 TR ID + 추정값 경로로 장중 5분마다 호출 중. 설계 미확정 상태에서 매 5분마다 잘못된 API 호출.
