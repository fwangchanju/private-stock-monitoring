## 문서 역할

이 문서는 사용자가 리눅스 서버에서 직접 수행해야 하는 작업만 정리한다.
설계 방향이나 미확정 설계 사안은 `plan.md`에서 관리한다.

## 서버 / 인프라 작업

- Docker / Docker Compose 실행 환경 준비
- nginx 컨테이너 구성
- backend 컨테이너 구성
- mariadb 컨테이너 구성
- 컨테이너 간 네트워크 및 기동 순서 확인
- 데이터 저장 디렉토리 / 볼륨 경로 정리
- 로그 파일 및 디스크 사용량 관리 방식 반영

## nginx / 접근 경로 작업

- 외부 진입용 대시보드 URL 결정
- nginx reverse proxy 설정
- `/api` 경로를 backend로 프록시하도록 설정
- 정적 프론트와 API 경로가 충돌하지 않도록 확인
- 필요 시 HTTPS / 인증서 적용 여부 검토 및 반영

## MariaDB 작업

- PSMS용 DB 생성
- 계정 및 권한 생성
- 접속 정보 정리
- 백업/복구 기본 방식 메모

## 환경변수 / 시크릿 작업

- 키움 API 인증정보 주입 방식 정리
- Spring 환경변수 파일 구성
- base URL 설정
- 운영/개발 환경값 분리 여부 확인

## 텔레그램 작업

- 메시지에 포함할 대시보드 링크 실제 동작 확인

## 운영 확인 작업

- 서버 재기동 후 컨테이너 자동 기동 여부 확인
- nginx → backend 연결 확인
- backend → mariadb 연결 확인
- 외부에서 대시보드 URL 접속 확인
- 텔레그램 링크 클릭 시 정상 진입 확인
