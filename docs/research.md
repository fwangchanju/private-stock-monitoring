# Research

이 문서는 참고 사이트의 화면 구성과 기능적 특징을 분석하기 위한 문서이다.

이 문서의 목적은 특정 사이트를 그대로 복제하는 것이 아니라,
어떤 기능 묶음과 화면 흐름이 실제 서비스에서 사용되는지 파악하고,
현재 `stock-dashboard-service`에 어떤 부분을 참고할지 정리하는 데 있다.

현재 프로젝트 기준으로는
**국내시장 정시 체크용 보조 웹 대시보드**라는 목적에 맞는 요소만 추려서 참고한다.

## 2. LongShortNow 분석

### 서비스 성격

LongShortNow는 단순 시세 조회 사이트라기보다,
주식과 코인에 대한 **실시간 예측 + 종목 토론 + 커뮤니티** 성격이 강한 서비스로 보인다.

즉 이 서비스의 중심은 데이터 대시보드 자체보다는
**시장 정보와 사용자 반응을 함께 소비하는 커뮤니티형 투자 서비스**에 가깝다.

### 핵심 기능

#### 2-1. 실시간 예측
사용자가 특정 종목이나 코인에 대해 상승/하락 관점의 예측을 확인하거나 참여하는 기능이 핵심으로 보인다.

#### 2-2. 종목별 토론 / 커뮤니티
국내·해외 주식 종목별 토론방, 코인별 토론, 일반 커뮤니티를 제공하는 방향으로 보인다.

#### 2-3. 주식 + 코인 동시 취급
LongShortNow는 주식과 코인을 함께 다루며,
하나의 서비스 안에서 서로 다른 자산군을 함께 소비하게 만드는 방향성이 있다.

### 화면/제품 관점에서 참고할 점

- 단순 시세판보다 **관심 자산을 빠르게 훑는 진입 화면**이 중요해 보인다.
- 데이터만 나열하는 구조보다, **시장 상황을 한눈에 보는 섹션형 구성**이 어울린다.
- 자산군이 달라도 같은 UX 안에서 소비되도록 묶는 방식은 참고할 수 있다.

### 참고하되 그대로 가져오지 않을 점

- 예측 참여 기능
- 종목 토론 커뮤니티
- 자유 게시판
- 소셜 기능 중심 구조

LongShortNow의 차별점은 커뮤니티에 강하게 실려 있으므로,
현재 프로젝트처럼 **정시 체크용 데이터 대시보드**를 만들 때는 이 부분을 그대로 가져오지 않아도 된다.

## 3. Prober 분석

### 서비스 성격

Prober는 공개적으로 보이는 메뉴 구조상,
**실시간 금융 데이터 탐색 플랫폼** 성격이 강하다.

LongShortNow보다 커뮤니티보다는 데이터 탐색 기능이 전면에 나오며,
시장 리스트 / 히트맵 / 자산군 분리 / 경제 캘린더 같은 요소를 묶어 두는 방향으로 보인다.

### 공개적으로 확인되는 메뉴/기능

공개 노출 기준으로 아래와 같은 흐름을 참고할 수 있다.

- 홈
- 커뮤니티
- 한국주식
- 히트맵
- 미국주식
- 암호화폐
- 경제캘린더
- 기타 확장 영역

즉 Prober는 **데이터 중심 대시보드/터미널형 서비스**에 더 가깝다.

### 핵심 기능

#### 3-1. 실시간 종목 리스트
국내 주식 시세를 리스트 형태로 탐색하게 하는 기능이 핵심 중 하나로 보인다.

#### 3-2. 히트맵
많은 종목을 한 화면에서 시각적으로 보게 하는 히트맵은
시장 전체 분위기를 빠르게 파악하게 하는 장치로 보인다.

#### 3-3. 멀티 자산 데이터 묶음
한국주식, 미국주식, 암호화폐 등 여러 자산군을 함께 배치하여
하나의 서비스 안에서 탐색하게 하는 방향이 보인다.

#### 3-4. 경제 캘린더 / 보조 정보
단순 시세판을 넘어서,
투자 판단에 필요한 주변 정보까지 함께 제공하려는 성격이 있다.

### 화면/제품 관점에서 참고할 점

- **리스트형 시세 화면**은 종목 탐색에 직접적이다.
- **히트맵**은 시장 전체 분위기를 한눈에 파악하게 해준다.
- 카테고리 분리(한국주식 / 미국주식 / 암호화폐)는 사용자가 자산군을 전환하며 탐색하기 좋다.
- 메인 서비스가 커뮤니티보다 **데이터 탐색 중심**으로 보인다.

### 참고하되 그대로 가져오지 않을 점

- 멀티 자산 전체를 처음부터 한 번에 지원하는 구조
- 경제 캘린더, 부동산 등 주변 정보까지 넓게 확장된 범위
- 대규모 전체 시장 히트맵을 위한 무거운 데이터 수집 구조

현재 프로젝트는 국내시장 중심이므로,
Prober의 전체 범위를 그대로 따라가기보다는
**시장 탐색 방식만 선택적으로 참고**하는 것이 적절하다.

## 4. TradingView 분석

### 서비스 성격

TradingView는 단순 차트 사이트가 아니라,
**차트 + 시장 요약 + 스크리너 + 히트맵 + 캘린더 + 커뮤니티 아이디어**를 함께 제공하는 종합형 시장 탐색 서비스에 가깝다.

한국 사이트 기준으로도 상단 메뉴에
`프로덕트 / 커뮤니티 / 마켓 / 브로커 / 더보기` 구조가 있고,
메인/마켓 계열 화면에서는 주요 지수, 한국 주식, 거래량 상위, 변동성 상위, 상승/하락 주식, 실적 캘린더 같은 섹션형 진입을 제공한다.

즉 TradingView는 단순히 “차트를 보는 곳”이 아니라,
**시장을 탐색하고, 후보를 찾고, 차트로 파고들고, 다른 사람의 아이디어까지 소비하는 플랫폼**으로 보는 것이 더 정확하다.

### 공개적으로 확인되는 핵심 제품/기능

#### 4-1. 수퍼차트
TradingView는 Supercharts를 핵심 제품으로 두고 있다.
공식 지원 문서 기준으로 Supercharts는 기술적 데이터, 재무 데이터, 커뮤니티 아이디어, 주요 경제 이벤트, 브로커 연결까지 하나의 작업 공간에서 다루도록 설계되어 있다.

즉 차트 자체가 독립 기능이 아니라,
다른 탐색 기능과 연결되는 중심 허브 역할을 한다.

#### 4-2. 시장 요약 / 마켓 페이지
한국 사이트의 마켓 페이지에서는
국내 지수, 한국 주식, 거래량 상위, 변동성 상위, 상승 주식, 하락 주식, 실적 캘린더, IPO 캘린더 등
“오늘 무엇을 먼저 볼 것인가” 관점의 섹션형 구성이 확인된다.

이 구조는 사용자가 특정 종목을 모르더라도
시장 전체에서 눈에 띄는 항목을 빠르게 찾게 해 준다.

#### 4-3. 스크리너
공식 기능 페이지 기준으로 TradingView는
주식 / 채권 / ETF / 암호화폐 / CEX / DEX 등 여러 스크리너를 제공한다.

특히 주식 스크리너는
배당수익률, 재무 배수, 성과, 기술적 지표 등 다양한 필터로 종목을 찾게 하는 구조다.

즉 스크리너는 단순 정렬 테이블이 아니라,
**조건 검색 + 후보 압축 도구** 역할을 한다.

#### 4-4. 히트맵
공식 기능 페이지 기준으로 히트맵은
주식 / ETF / 암호화폐를 한눈에 조망하게 하며,
섹터별 그룹화, 필터, 사용자 정의 저장/공유, 간단 통계 확인 기능을 제공한다.

즉 히트맵은 종목 하나하나를 읽는 화면이 아니라,
**시장 전체를 빠르게 스캔하는 시각 도구**다.

#### 4-5. 캘린더
공식 기능 페이지 기준으로 TradingView는
경제 이벤트, 실적, 배당 정보를 Calendar 안에서 함께 제공한다.

이는 가격 데이터만으로 끝나지 않고,
시장 이벤트 일정 자체를 하나의 탐색 축으로 다루고 있다는 의미다.

#### 4-6. 커뮤니티 아이디어
TradingView는 아이디어 페이지를 통해
사용자 분석 글, 롱/숏 관점, 차트 해설, 에디터즈 픽 등을 노출한다.

즉 커뮤니티가 있긴 하지만,
LongShortNow처럼 토론방 중심이라기보다는
**차트 기반 아이디어 발행/소비 구조**에 더 가깝다.

#### 4-7. Pine Script / 커스텀 지표
TradingView는 Pine Script를 통해
사용자가 직접 인디케이터/전략을 작성하고 차트에 적용할 수 있게 한다.

이 부분은 일반 데이터 대시보드보다 훨씬 확장된 영역이며,
TradingView가 “플랫폼”으로 평가받는 이유 중 하나다.

#### 4-8. 재무 데이터 접근
공식 지원 문서 기준으로 Supercharts의 Financials 탭에서
손익계산서, 대차대조표, 현금흐름 데이터에 접근할 수 있다.

즉 TradingView는 기술적 분석뿐 아니라,
기초 재무 정보 접근도 한 제품 흐름 안에 포함시킨다.

### 화면/제품 관점에서 참고할 점

#### A. 섹션형 메인 구성
TradingView의 마켓 페이지는
“시장 전체 → 거래량 상위 → 변동성 상위 → 상승/하락 → 이벤트”처럼
사용자가 **탐색 순서 자체를 따라가게 만드는 구조**가 강하다.

이건 현재 프로젝트 메인 화면을 설계할 때 참고 가치가 높다.

예를 들어 현재 프로젝트에서도
- 시장종합
- 투자자별 매매종합
- 장중 투자자별 매매 상위
- 프로그램매매 / 공매도 핵심 표
처럼 **정시 체크 순서에 맞는 섹션 배치**를 할 수 있다.

#### B. 표 + 차트 + 요약의 조합
TradingView는 차트 사이트로 알려져 있지만,
실제로는 차트만 던져두지 않고 리스트 / 랭킹 / 요약 블록을 함께 사용한다.

현재 프로젝트도 차트 중심 서비스는 아니지만,
필요하다면 특정 표의 상세 화면에서 최소한의 추이 시각화를 붙이는 방향은 검토할 수 있다.

#### C. 스크리너적 사고방식
TradingView의 스크리너는 단순 정렬표가 아니라
사용자가 조건으로 후보를 좁히는 도구다.

현재 프로젝트는 처음부터 범용 스크리너를 만들 필요는 없지만,
이후 확장 시 아래 정도는 참고 가능하다.

- 투자자 순매수 상위만 보기
- 프로그램 순매수 강한 종목만 보기
- 공매도 비중이 특정 기준 이상인 종목만 보기
- 특정 시장/섹터만 필터링해서 보기

즉 현재 프로젝트의 표들도 장기적으로는
**정적 표 → 조건 기반 탐색 표**로 발전할 수 있다.

#### D. 히트맵의 사용 목적
TradingView의 히트맵은 종목 상세 분석용이라기보다,
**시장 전체의 압축 시각화**에 가깝다.

현재 프로젝트에서도 당장 대규모 전체시장 히트맵을 만들 필요는 없지만,
향후 “시장종합” 또는 “장중 강세/약세 군집”을 보여주는 보조 시각화 아이디어로는 참고할 수 있다.

#### E. 캘린더성 정보의 위치
TradingView는 캘린더를 핵심 제품군으로 두지만,
현재 프로젝트에서는 범위를 과도하게 넓히지 않는 것이 더 중요하다.

다만 향후 확장 시
- 휴장일
- 옵션만기일
- 주요 일정
정도의 **국내시장 체크 보조 캘린더**는 검토 가치가 있다.

#### F. 롱/숏 뷰 관점에서 참고할 점
TradingView에는 롱/숏 아이디어를 소비하는 흐름과,
일부 자산군에서는 Long/Short Ratio 같은 심리성 지표도 존재한다.

다만 현재 프로젝트는
**국내시장 정시 체크용 표 기반 대시보드**가 목적이므로,
초기 단계에서 롱/숏 의견 커뮤니티나 심리지표를 핵심 기능으로 가져올 필요는 없다.

현 단계에서 더 참고할 가치가 큰 것은
“롱/숏 의견 자체”보다도
**시장 후보를 빠르게 추리는 진입 구조와 탐색 UX**다.

### 참고하되 그대로 가져오지 않을 점

- 범용 글로벌 멀티자산 플랫폼 구조 전체
- 사용자 아이디어/롱숏 의견 커뮤니티
- Pine Script 수준의 커스텀 지표 플랫폼
- 브로커 연동 / 실제 거래 기능
- 방대한 범용 스크리너 전체 기능
- 대규모 글로벌 캘린더/재무 데이터 범위

현재 `stock-dashboard-service`의 목적은
**국내시장 정시 체크용 보조 대시보드**이므로,
TradingView처럼 모든 것을 한 서비스 안에 넣는 방향은 맞지 않다.

현재 단계에서 가져올 가치가 큰 것은 아래다.

- 메인 화면의 섹션형 탐색 구조
- 시장을 빠르게 훑는 요약/랭킹 구도
- 표를 나중에 필터 가능한 구조로 발전시킬 수 있다는 사고방식
- 필요 시 최소한의 추이 시각화 결합 아이디어

---

## 5. LongShortNow / Prober / TradingView 비교

### 공통점

- 모두 금융/투자 관심 사용자를 대상으로 한다.
- 모두 “지금 무엇을 먼저 볼 것인가”를 빠르게 보여주는 진입 구조가 중요하다.
- 단순 종목 상세 하나보다, **시장 전체를 훑는 진입 화면**이 강하다.

### 차이점

#### LongShortNow
- 실시간 예측, 종목 토론, 커뮤니티 중심
- 사용자 반응과 참여가 핵심
- 소셜형 투자 서비스 성격이 강함

#### Prober
- 실시간 시세, 리스트, 히트맵, 캘린더 중심
- 데이터 탐색과 시장 모니터링이 핵심
- 대시보드/터미널형 성격이 더 강함

#### TradingView
- 차트, 마켓 탐색, 스크리너, 히트맵, 캘린더, 커뮤니티 아이디어까지 결합
- 차트 허브 중심의 종합 플랫폼 성격
- 단일 기능 서비스가 아니라 **탐색 → 분석 → 아이디어 소비**를 묶은 구조

## 6. 현재 프로젝트 관점에서의 참고 결론

현재 프로젝트는
**국내시장 핵심 표를 웹으로 재구성하고, 정시 리마인더로 확인을 놓치지 않게 하는 서비스**다.

이 기준에서 참고 우선순위는 아래와 같다.

### 6-1. 가장 많이 참고할 대상
#### Prober + TradingView의 데이터 탐색 구조
가장 유효한 참고 방향은
커뮤니티보다 **데이터 탐색형 대시보드 구조**다.

구체적으로는 아래가 유효하다.

- 메인에서 시장을 빠르게 훑게 하는 섹션형 배치
- 랭킹 / 요약 / 시세 리스트 중심 구조
- 시장 전체를 스캔하는 시각 요소
- 표별 상세 진입 구조

### 6-2. 부분적으로만 참고할 대상
#### LongShortNow의 진입 화면 감각
LongShortNow의 커뮤니티 기능은 현재 프로젝트와 맞지 않지만,
사용자가 관심 자산이나 현재 뜨는 흐름을 빠르게 훑게 만드는 진입 감각은 일부 참고 가능하다.

### 6-3. 현재 프로젝트에서 명확히 제외할 방향
- 커뮤니티 / 토론방 중심 서비스
- 롱/숏 투표 기능
- 글로벌 멀티자산 플랫폼 전체 범위
- Pine Script형 사용자 확장 플랫폼
- 범용 브로커 연동
- 주문/매매 기능

## 7. 현재 프로젝트에 실제로 반영해볼 수 있는 아이디어

### 아이디어 1. 메인 화면을 “정시 체크 순서” 기반 섹션형으로 구성
예시
- 시장종합
- 투자자별 매매종합
- 장중 투자자별 매매 상위
- 프로그램매매 관련 표
- 공매도 관련 표
- 마지막 갱신 시각

이 방향은 TradingView의 마켓 탐색 구조와 잘 맞는다.

### 아이디어 2. 표를 단순 조회가 아니라 “후보 탐색” 구조로 확장 가능하게 설계
초기에는 단순 표로 시작하더라도,
나중에는 필터를 붙일 수 있는 구조를 생각해둘 수 있다.

예시
- 투자자 구분 필터
- 시장 구분 필터
- 순매수 기준치 필터
- 공매도 비중 기준 필터

### 아이디어 3. 상세 화면에는 최소한의 시각화 여지 남기기
현재 프로젝트는 차트 서비스가 아니지만,
종목별 프로그램매매추이 / 공매도추이 같은 화면은
향후 간단한 추이 시각화와 결합할 여지가 있다.

### 아이디어 4. “시장 전체 훑기”용 보조 시각화는 장기 검토
히트맵 전체 구현은 초기 범위에 과할 수 있지만,
시장종합 화면에서
- 상승/하락 분포
- 주요 섹터 강약
- 강세/약세 묶음
같은 압축 시각화는 장기적으로 검토할 수 있다.

## 8. 최종 정리

현재 프로젝트에서 가장 중요한 것은
TradingView처럼 모든 기능을 다 넣는 것이 아니라,
**사용자가 정해진 시각에 들어와서 국내시장 핵심 표를 빠르게 확인하고 나갈 수 있게 만드는 것**이다.

따라서 참고 방향은 아래처럼 정리할 수 있다.

### 가져올 것
- 섹션형 메인 구조
- 시장 훑기 중심 진입 방식
- 요약 + 랭킹 + 상세 진입 구조
- 향후 필터 확장 가능성을 고려한 표 설계

### 보류하거나 제외할 것
- 커뮤니티/소셜 기능
- 사용자 아이디어 발행 구조
- 고급 차트 플랫폼 전체
- 범용 글로벌 데이터 플랫폼 확장
- 주문/거래 기능

즉 현재 프로젝트의 가장 적절한 참고 축은
**LongShortNow의 소셜성**이 아니라,
**Prober와 TradingView의 시장 탐색형 구조**라고 정리할 수 있다.
